## 1. 严格按设计实现 — ❌ 不通过

### 1.1 Classroom.RoomType — 字段不存在于设计

**设计文档 §3.1.1** 规定 Classroom 追加字段为 `Equipment`（不是 `RoomType`）。

**设计文档 §2.2.4** `Match()` 函数使用 `room.Type`（现有字段）做匹配，不使用 `room.RoomType`。

**后端模型** `classroom.go` 有 `Equipment` 字段，**没有** `RoomType` 字段。

**前端实现** 在 `types/index.ts` 定义了 `Classroom.roomType?: string`，在 `ResourcePage.vue` 添加了 `roomType` 表单字段和表格列。

**后果**：前端编辑 `roomType` 后通过 `RS.UpdateClassroom(m)` 发送到后端，GORM 忽略不存在的字段，**数据不会持久化**。用户填写后保存，重新打开编辑窗会看到空值。

**严重级别**: 🔴 Critical

### 1.2 Course.Category 选项值域 — 与后端常量不一致

**设计文档 §3.3.2** 规定 category 选项值为英文：
```
theory / lab / pe / computer / seminar / art
```

**后端常量** `room_types.go` `CourseCategory`：
```
CategoryTheory = "theory"
CategoryLab = "lab"
CategoryPE = "pe"
CategoryComputer = "computer"
CategorySeminar = "seminar"
CategoryArt = "art"
```

**前端实现** `categoryOptions` 值为中文：
```
普通 / 实验 / 上机 / 体育 / 外语 / 艺术
```

**后果**：
- 前端保存的 `category` 值（如"体育"）无法匹配后端 `CategoryRoomTypeMap`（key 为 `"pe"`）
- `InferRoomType()` 对前端保存的 category 值返回空字符串（映射查不到），退化为名称推断
- 违反"单一真相源"原则

**严重级别**: 🔴 Critical

### 1.3 RoomType 选项不完整

**后端常量** `room_types.go` 定义 6 种 RoomType：
```
普通教室 / 多媒体教室 / 实验室 / 机房 / 体育馆 / 阶梯教室
```

**前端实现** `roomTypeOptions` 只有 5 种，**缺少** `多媒体教室` 和 `阶梯教室`，**多出** `语音室`（后端不存在）。

**设计文档 §3.3.2** 教学任务 requiredRoomType 选项明确列出了 6 种 + "不指定"。

**后果**：
- 用户无法选择"多媒体教室"或"阶梯教室"
- 用户选择了"语音室"，后端无法识别，匹配逻辑失效

**严重级别**: 🟡 Major

### 1.4 Course.Category 缺少 "研讨课" 选项

**设计文档** 和后端常量包含 `seminar`（研讨课），前端 `categoryOptions` 没有。

**设计文档** 和后端常量**不包含** "外语"，前端 `categoryOptions` 多了"外语"。

**严重级别**: 🟡 Major

## 2. Scope Creep — ⚠️ 存在偏差

### 2.1 Equipment 字段未实现

**设计文档 §3.3.2** 规定教室编辑表单应增加设备字段（tags 输入）。

**实施计划 P6** 步骤 2 明确要求："教室表单追加设备字段（tags 输入）"。

**MVP 约定**（用户确认）：仅围绕 RoomType 做 URMF，Equipment/RequiredEquipment 全部移除。

**判定**：Equipment 未实现符合用户确认的 MVP 约定，但与设计文档/实施计划不一致。**这不是 Scope Creep，而是 MVP 裁剪**，可接受。但 `Classroom.roomType` 是前端自行添加的字段，**不在设计文档中**，属于 Scope Creep。

### 2.2 Classroom.roomType — Scope Creep

`Classroom.roomType` 字段不在设计文档中，不在实施计划中。设计文档的匹配逻辑使用现有的 `Classroom.Type` 字段。前端新增 `roomType` 是超范围开发。

**严重级别**: 🔴 Critical

## 4. UI/交互问题 — ⚠️ 存在问题

### 4.1 教室编辑表单有冗余字段

教室表单现在有两个类型相关字段：
- `type`（类型（选填））— 原有字段
- `roomType`（教室类型）— 新增字段

两者语义重叠，用户困惑：应该填哪个？设计文档中没有 `roomType` 字段，匹配逻辑用 `type`。

**严重级别**: 🟡 Major

### 4.2 教学任务 requiredRoomType 列渲染样式

使用 `var(--b3-theme-on-surface-light)` CSS 变量，但 Naive UI 主题系统不一定定义此变量。应使用 Naive UI 的 `NText` 组件或 `opacity` 样式。

**严重级别**: 🟢 Minor

### 4.3 教学任务 requiredRoomType 选项来源

`requiredRoomTypeOptions` 复用 `roomTypeOptions` + `{ label: '不指定（自动推断）', value: '' }`。由于 `roomTypeOptions` 本身值域错误（见 1.3），`requiredRoomTypeOptions` 也继承了错误。

1. 移除 `Classroom.roomType` 相关的所有代码（类型定义、表格列、表单字段）
2. 将 `categoryOptions` 值改为英文（与后端 `CourseCategory` 常量一致）
3. 将 `roomTypeOptions` 值改为后端 6 种 RoomType 常量（移除"语音室"，补充"多媒体教室"和"阶梯教室"）
4. `requiredRoomTypeOptions` 随 F3 自动修正
