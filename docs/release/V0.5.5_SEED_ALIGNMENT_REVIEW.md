## 一、当前 Seed 创建的时间相关数据

### 1.1 Seed 数据清单

| Seed 函数 | 创建的实体 | 数量 | 时间相关字段 |
|---|---|---|---|
| `seedBaseData` | `Semester` | 3 | `Name`（含学年+学期文字）、`IsActive`（布尔） |
| `seedTeachingTasks` | `TeachingTask` | 34 | `SemesterID`（FK）、`StartWeek=1`、`EndWeek=16`（硬编码） |
| `seedTeachingTasks` | `TeachingTaskClass` | 34 | `TeachingTaskID`、`ClassGroupID`（与时间无关） |

| - | `ScheduleVersion` | 0 | 不 seed（运行时生成） |

### 1.2 关键发现：Seed 未设置 `Semester.StartDate`

```go
// seed.go:57-61
semesters := []models.Semester{
    {Name: "2025-2026 第二学期", IsActive: true},
    {Name: "2025-2026 第一学期", IsActive: false},
    {Name: "2024-2025 第二学期", IsActive: false},
}
```

**`StartDate` 字段为空字符串**。这意味着当前系统实际上**没有任何日期锚点**，前端 `WeekView.vue` 不得不使用"当年 1 月 1 日 + week×7"伪计算日期。

## 三、未来 Semester 模型与 Seed 调整需求

### 3.1 Phase 1 目标模型

```go
type Semester struct {
    gorm.Model
    AcademicYear  string    // "2025-2026"
    Term          string    // "FIRST" | "SECOND"
    StartDate     time.Time // 周一
    TeachingWeeks int       // 21
}
```

**删除**：`Name`、`IsActive`、`string StartDate`

### 3.2 Seed 调整需求矩阵

| 当前 Seed 行为 | Phase 1 后调整方向 |
|---|---|
| `Name: "2025-2026 第二学期"` | 拆为 `AcademicYear: "2025-2026"` + `Term: "SECOND"` |
| `IsActive: true`（标记活跃学期） | 删除。改为 `CurrentSemester(semesters, now)` 按时间派生 |
| `StartDate` 空字符串 | 必须填入真实日期，且为周一：`2026-02-23`（第二学期） |
| `TeachingWeeks` 不存在 | 新增：`21`（第一学期）/ `22`（第二学期） |
| `seedIfAbsent` 用 `name` 做唯一键 | 改为 `(academic_year, term)` 复合唯一键 |
| `seedTeachingTasks` 用 `IsActive` 选活跃学期 | 改为 `CurrentSemester(semesters, now)` 或固定选第一个学期 |
| `seedDemoEntries` 硬编码 `Semester: "2025-2026 第二学期"` | 改为运行时查询 SemesterID 后赋值 `SemesterID: activeSemesterID` |
| `StartWeek: 1, EndWeek: 16` | 暂不动（Phase 2 配合 AcademicTerm 校准为 1-18） |
| `Weeks: "1-16"` | 暂不动（Phase 2 校准） |

### 3.3 新 Seed 数据设计

```go
semesters := []models.Semester{
    {
        AcademicYear:  "2025-2026",
        Term:          "SECOND",
        StartDate:     time.Date(2026, 2, 23, 0, 0, 0, 0, time.UTC), // 周一
        TeachingWeeks: 22,
    },
    {
        AcademicYear:  "2025-2026",
        Term:          "FIRST",
        StartDate:     time.Date(2025, 9, 1, 0, 0, 0, 0, time.UTC), // 周一
        TeachingWeeks: 21,
    },
    {
        AcademicYear:  "2024-2025",
        Term:          "SECOND",
        StartDate:     time.Date(2025, 2, 24, 0, 0, 0, 0, time.UTC), // 周一
        TeachingWeeks: 22,
    },
}
// 唯一键：(academic_year, term)
seedIfAbsent(db, semesters, "academic_year = ? AND term = ?",
    func(s models.Semester) []interface{} { return []interface{}{s.AcademicYear, s.Term} })
```

**注意**：`seedIfAbsent` 当前只支持单字段唯一键。Phase 1 需扩展为支持复合键，或改用直接 `FirstOrCreate` 调用。

### 3.4 `seedTeachingTasks` 调整

```go
// 当前（依赖 IsActive）
activeSemesterID := uint(0)
for _, s := range semesters {
    if s.IsActive {
        activeSemesterID = s.ID
        break
    }
}

// Phase 1 后（按时间派生 或 固定选第一个）
// 方案 A: 按 CurrentSemester(now) 派生
activeSemesterID := academic_calendar.CurrentSemester(semesters, time.Now()).ID

// 方案 B: 固定选第一个（最简单，seed 不依赖时间）
activeSemesterID := semesters[0].ID
```

**建议方案 B**：seed 是演示数据，不需要按时间智能选择。固定选第一个（最新的学期）即可。

### 3.5 `seedDemoEntries` 调整

```go
// 当前（硬编码字符串）
entries := []models.ScheduleEntry{
    {CourseID: 18, ..., Semester: "2025-2026 第二学期", ...},
}

// Phase 1 后（运行时查询 SemesterID）
// 需先查出 activeSemesterID，再赋值
entries := []models.ScheduleEntry{
    {CourseID: 18, ..., SemesterID: activeSemesterID, ...},
}
```

`seedDemoEntries` 需接收 `activeSemesterID` 参数，或内部重新查询。

## 四、与 Phase 1 Semester Refactor 的关系

### 4.1 依赖关系图

```
Phase 1 Semester Refactor
    │
    ├── 1. models/setting.go: Semester 模型重写
    │       └── 依赖: seed.go 必须同步更新（否则编译失败）
    │
    ├── 2. models/schedule_entry.go: Semester string -> SemesterID uint
    │       └── 依赖: seed.go seedDemoEntries 必须同步
    │
    ├── 3. services/resource_service.go: 删除 IsActive 逻辑
    │       └── 依赖: seed.go 不再使用 IsActive
    │
    ├── 4. services/scheduling_service.go: SchedulingConfig.Semester string 删除
    │       └── 依赖: seed.go 不受直接影响
    │
    ├── 5. services/version_service.go: 删除 resolveSemesterID(name)
    │       └── 依赖: seed.go 不受直接影响
    │
    └── 6. seed.go: 全部重写 Semester 相关 seed
            └── 这是 Phase 1 的必要同步项
```

### 4.2 Seed 修改是 Phase 1 的硬性同步项

Phase 1 修改 Semester 模型后，seed.go **必须同步修改**，否则：
- 编译失败（`Name`、`IsActive` 字段不存在）
- 运行时失败（`seedIfAbsent` 用 `name` 做唯一键，字段已删）
- 数据错误（`seedDemoEntries` 硬编码字符串 `"2025-2026 第二学期"` 无对应字段）

**结论**：seed.go 修改不是独立任务，而是 Phase 1 实施步骤的一部分。

## 八、风险

| 风险 | 概率 | 缓解 |
|---|---|---|
| `seedIfAbsent` 泛型函数不支持复合键 | 高 | 改为直接 `FirstOrCreate` 调用，不走 `seedIfAbsent` |
| `seedDemoEntries` 需要 SemesterID 但当前不接收参数 | 中 | 改为接收 `semesterID uint` 参数，或内部查询 |

| `TeachingWeeks` 值与未来 AcademicTerm 不一致 | 低 | Phase 2 校准，Phase 1 仅填入合理值 |
