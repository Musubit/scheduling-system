## 3. v0.6.0 需要完成的工作

### 3.1 双模式闭环 — 消除 INV-E1 违规

**当前问题**: TIME_ONLY 模式通过两个 hack 绕过教室分配：
1. `buildVirtualClassroomsForTimeOnly()` — 为每个 TeachingTask 创建虚拟教室（容量=学生数，类型=任务的 RequiredRoomType）
2. `applySyntheticClassroomIDsForTimeOnly()` — 标记 `entry.IsVirtual = true`

这导致 `schedule_entries` 表在 TIME_ONLY 下仍然有行——违反设计 spec INV-E1（要求 TIME_ONLY 下 schedule_entries 零行）。

**当前 ScheduleEntry 模型（旧，v0.5.4 风格）**:
```go
type ScheduleEntry struct {
    gorm.Model
    CourseID    uint      // ← 时间数据嵌在 Entry 里
    TeacherID   uint      // ← 时间数据嵌在 Entry 里
    ClassroomID uint      // ← not null，TIME_ONLY 被迫写合成 ID
    ClassGroupID   *uint
    TeachingTaskID *uint
    SemesterID  uint
    DayOfWeek   DayOfWeek  // ← 时间数据嵌在 Entry 里
    StartPeriod Period     // ← 时间数据嵌在 Entry 里
    Span        int        // ← 时间数据嵌在 Entry 里
    Weeks       string
    IsVirtual   bool       // ← TIME_ONLY hack 标记
    // + Associations (Course, Teacher, Classroom, ClassGroup, TeachingTask, Semester)
}
```

**目标模型（spec 设计）**:
```
TeachingTask → TimeAssignment → ScheduleEntry (+ Classroom, 仅 FULL)
```
- TIME_ONLY: 只写 TimeAssignment，schedule_entries 零行
- FULL: TimeAssignment + ScheduleEntry 一一对应

**已存在的 TimeAssignment 模型（未在生产路径使用）**:
```go
type TimeAssignment struct {
    ID                uint
    SemesterID        uint
    ScheduleVersionID uint
    TeachingTaskID    uint
    DayOfWeek         DayOfWeek
    StartPeriod       Period
    Span              int
    // 无 ClassroomID / TeacherID / CourseID
}
```
表已通过 AutoMigrate 创建，但注释明确说 "尚未被生产写路径使用"。

**决策阻塞**: 需要选择修复方案：
- 方案 A: ClassroomID 改 `*uint` nullable — 改动较小，但仍是耦合结构
- 方案 B: 拆 `time_assignments` + 重构 `schedule_entries` — 符合 spec，改动大

### 3.2 SchedulingOrchestrator 正式接管

**当前状态**: `SchedulingService.RunScheduling()` 约 600+ 行，包含：
- 数据加载（teachingTasks, classrooms, teachers, classGroups）
- 锁定时段合并
- SA 配置
- OR-Tools + SA 并行调度
- 评分比较
- 虚拟教室 hack
- 持久化
- 快照/版本创建
- 冲突分析

**目标**: 迁移到 `SchedulingOrchestrator` 装配模式：
```
SchedulingService (Facade: 加载数据 + 持久化)
        |
SchedulingOrchestrator (纯计算: 按 Mode 装配)
        |
+-------+-------+
|               |
ITimeScheduler  IRoomScheduler (仅 FULL)
```

**已有但未接入的 Orchestrator** (`backend/scheduling/orchestrator/orchestrator.go`, 108 行):
- 持有 `ITimeScheduler` + `IRoomScheduler` + `IScorer` 三个 interface
- `Run()` 方法实现完整的 Stage 1→2→3 流程
- 按 `Mode.RequiresRoomAssignment()` 决定是否跳过 Stage 2
- 不与数据库/模型层耦合（纯计算）
