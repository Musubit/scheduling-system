## 三、TeachingTask Domain Contract 检查

### 当前 TeachingTask 模型边界

| 概念 | 状态 | 说明 |
|---|---|---|
| 自动推断教学关系 | ❌ 已删除 | `DetectMergeableTasks`、`SplitMergedTeachingTask`、前端合班 UI 全部移除 |
| 显式多班绑定 | ✅ 保留 | `CreateTeachingTask(task, classGroupIDs []uint)` 签名不变 |
| `TeachingTaskClass` 多对多表 | ✅ 保留 | 数据和关联完整 |
| `TeachingTask.Classes` 关联字段 | ✅ 保留 | Go 结构体中 `Classes []TeachingTaskClass` 不变 |

### TeachingTask 当前服务方法

```
TeachingTaskService
├── CreateTeachingTask(task, classGroupIDs []uint)
├── UpdateTeachingTask(id, task, classGroupIDs []uint)
├── DeleteTeachingTask(id)
├── ListTeachingTasks(semesterID)
├── GetTeachingTask(id)
├── ImportTeachingTasks(semesterID, rows)
└── ExportTeachingTasks(semesterID)
```

### TeachingTask 生命周期（v0.5.4 后边界）

```
创建：
  表单创建：勾选多个班级 → CreateTeachingTask(task, [CG1, CG2, CG3])
  导入创建：CSV 行"CS2301,CS2302" → ImportTeachingTasks → 内部调 CreateTeachingTask

编辑：
  表单编辑：增减班级勾选 → UpdateTeachingTask(id, task, [CG1, CG2])

删除：
  删除任务同时 CASCADE 删除 TeachingTaskClass 关联

消费：
  Solver：tt.Classes → classIDs []uint → 教室/班级占用计算
  前端展示：列表中的班级多标签（NTag）
```

**结论**：多班绑定能力完整保留。删除的仅为"自动推测合班"路径。
