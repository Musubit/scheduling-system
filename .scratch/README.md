# .scratch

本地 Issue Tracker。每个功能一个子目录，结构如下：

```
.scratch/
├── README.md                  # 本文件
└── <feature-slug>/            # 功能目录
    ├── PRD.md                 # 产品需求文档
    └── issues/                # 实现 issue
        ├── 01-<slug>.md
        ├── 02-<slug>.md
        └── ...
```

## Issue 模板

```markdown
# <标题>

Status: 待评估

## 描述

...

## 验收标准

- [ ] ...

## Comments

```
