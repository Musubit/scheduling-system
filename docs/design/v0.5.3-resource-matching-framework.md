## 1. 问题分析

### 1.1 现状：散落的硬编码教室类型匹配

当前系统的资源匹配逻辑**散落在 7 个文件中**，全部基于课程名关键字推断：

| 位置 | 逻辑 |
|------|------|
| `roomtype.go` | `IsLabCourse(name)` 含"实验" / `IsComputerCourse(name)` 含"上机" |
| `models/course.go` | `IsSportsCourse(name)` 含"体育" |
| `sa_initial.go` L36-43 | `if IsSports → "体育馆" else if IsLab → "实验室" else if IsComputer → "机房"` |
| `sa_neighbors.go` L139-144 | 同上逻辑重复一遍 |
| `sa_postopt.go` L309-319 | 同上逻辑第三遍 |
| `sa_solver.go` L489-500 | `getRequiredRoomType()` 第四遍 |
| `scheduling_service.go` L506-513 | 第五遍 |
| `scheduler/solver.py` L404-416 | Python 侧 `SPECIALTY_ROOM_TYPES = {"体育馆", "实验室", "机房"}` 硬编码 |
| `move_service.go` | 无教室类型校验（**遗漏**：手动调课不检查目标教室类型） |

**问题清单**：
1. **5 处重复** — 同一段 if-else 链复制 5 遍，新增类型需改 5 处
2. **关键字耦合** — 课程类型靠名字猜（"实验"/"上机"/"体育"），无法配置
3. **遗漏** — `move_service.go` 的 `CheckMove` 不校验教室类型，调课时可以把实验课拖到普通教室
4. **不可扩展** — 想加"多媒体教室只排多媒体课"、"机房需要计算机"等约束无从下手
5. **SA 与 OR-Tools 规则不同步风险** — Go 侧改了 Python 侧不知道

### 1.2 目标

设计一套 **统一资源匹配框架 (URMF)**：
- **单一真相源** — 教室类型匹配规则只在一处定义
- **纯函数设计** — 无状态、无副作用，SA / OR-Tools / MoveService / 未来 ConstraintService 均可调用
- **结构化返回** — `MatchResult` 含 `OK` + `Code` + `Reason`，可复用于诊断、调课提示、日志、排课诊断报告
- **append-only** — 不修改现有模型字段语义，只追加新字段
- **Stable Core 零影响** — 不碰 `ScoreSchedule()` 签名和 7 个软约束
- **可扩展** — 支撑未来设备、校区、教师资质等维度

#### 2.2.2 EquipmentSet（`backend/models/equipment.go`）

```go
package models

import "encoding/json"

// EquipmentSet 封装设备集合，提供语义化查询方法。
// 存储格式: JSON array string，如 `["projector","smartboard","aircon"]`
// 空字符串 = 无设备。
type EquipmentSet struct {
    items map[string]bool
}

// ParseEquipment 从 JSON string 解析为 EquipmentSet。
// 解析失败返回空集合（不报错，降级处理）。
func ParseEquipment(jsonStr string) EquipmentSet {
    if jsonStr == "" {
        return EquipmentSet{}
    }
    var arr []string
    if err := json.Unmarshal([]byte(jsonStr), &arr); err != nil {
        return EquipmentSet{}
    }
    s := EquipmentSet{items: make(map[string]bool, len(arr))}
    for _, e := range arr {
        s.items[e] = true
    }
    return s
}

// Has 检查是否包含某个设备。
func (s EquipmentSet) Has(item string) bool {
    return s.items[item]
}

// ContainsAll 检查是否包含所有所需设备。
func (s EquipmentSet) ContainsAll(required EquipmentSet) bool {
    for item := range required.items {
        if !s.items[item] {
            return false
        }
    }
    return true
}

// IsEmpty 检查是否为空集。
func (s EquipmentSet) IsEmpty() bool {
    return len(s.items) == 0
}

// ToJSON 序列化回 JSON string。
func (s EquipmentSet) ToJSON() string {
    if len(s.items) == 0 {
        return ""
    }
    arr := make([]string, 0, len(s.items))
    for item := range s.items {
        arr = append(arr, item)
    }
    b, _ := json.Marshal(arr)
    return string(b)
}
```

#### 2.2.3 MatchResult（`backend/services/resource_matcher.go`）

```go
package services

import "scheduling-system/backend/models"

// ResourceMatchCode 标识匹配结果分类，用于 switch/日志/诊断。
// 使用 int 枚举而非 string，便于 switch 且避免拼写错误。
type ResourceMatchCode int

const (
    MatchOK             ResourceMatchCode = iota // 匹配通过
    CodeRoomTypeMismatch                         // 教室类型不匹配
    CodeEquipmentMissing                         // 缺少所需设备
    CodeSpecialtyExclusion                       // 排他教室类型冲突（普通课不能用实验室等）
)

// MatchResult 是所有匹配调用的统一返回类型。
// 不返回 bool — 调用方可直接使用 Code / Reason 做诊断。
//
// 使用方式:
//   if !result.OK { log.Warn(result.Code, result.Reason) }
//   switch result.Code { case CodeRoomTypeMismatch: ... }
type MatchResult struct {
    OK               bool               `json:"ok"`
    Code             ResourceMatchCode  `json:"code"`                       // 0=OK, >0=失败类型
    Reason           string             `json:"reason,omitempty"`           // 人类可读，可直接用于调课冲突提示
    RequiredType     string             `json:"requiredType,omitempty"`     // 期望的教室类型
    ActualType       string             `json:"actualType,omitempty"`       // 实际教室类型
    MissingEquipment []string           `json:"missingEquipment,omitempty"` // 缺少的设备列表
}

// ResourceRequirement 是 ExplainRequirement() 的返回，描述任务的完整资源需求。
// 纯结构化数据，不含展示文案 — 调用方自行组装展示。
// 用于排课诊断报告、调课失败提示、日志输出。
type ResourceRequirement struct {
    RoomType       string   `json:"roomType"`               // 推导出的教室类型（空=不限）
    RoomTypeSource string   `json:"roomTypeSource"`         // "explicit" | "category" | "name_fallback" | "none"
    Equipment      []string `json:"equipment"`              // 所需设备列表
    IsShared       bool     `json:"isShared"`               // 是否使用共享场地
}
```

#### 2.2.4 匹配函数（`backend/services/resource_matcher.go`）

```go
// ResourceMatcher 是无状态的工具函数集。
// 所有函数都是纯函数：相同输入永远产生相同输出，无副作用。
// 不查 DB、不评分、不记日志、不修改数据。
//
// 调用方负责数据加载，matcher 只负责判断。

// Match 检查教室是否满足任务的资源需求（硬约束）。
// 返回 MatchResult，包含失败原因，可直接用于冲突报告。
func Match(task models.TeachingTask, course models.Course, room models.Classroom) MatchResult {
    // 1. 教室类型检查
    reqType := InferRoomType(task, course)
    if reqType != "" {
        if room.Type != reqType {
            return MatchResult{
                OK:           false,
                Code:         CodeRoomTypeMismatch,
                Reason:       fmt.Sprintf("需要%s，当前教室为%s", reqType, room.Type),
                RequiredType: reqType,
                ActualType:   room.Type,
            }
        }
    } else {
        // 无显式需求 — 检查排他教室
        if models.SpecialtyRoomTypes[room.Type] {
            return MatchResult{
                OK:         false,
                Code:       CodeSpecialtyExclusion,
                Reason:     fmt.Sprintf("普通课程不能使用%s", room.Type),
                ActualType: room.Type,
            }
        }
    }

    // 2. 设备检查
    reqEquip := ParseEquipment(task.RequiredEquipment)
    if !reqEquip.IsEmpty() {
        roomEquip := ParseEquipment(room.Equipment)
        var missing []string
        for item := range reqEquip.items {
            if !roomEquip.Has(item) {
                missing = append(missing, item)
            }
        }
        if len(missing) > 0 {
            return MatchResult{
                OK:               false,
                Code:             CodeEquipmentMissing,
                Reason:           fmt.Sprintf("缺少设备: %s", strings.Join(missing, ", ")),
                MissingEquipment: missing,
            }
        }
    }

    return MatchResult{OK: true, Code: MatchOK}
}

// InferRoomType 推导任务所需的教室类型。
// 优先级: task.RequiredRoomType > Course.Category 映射 > 名称推断 (Deprecated)
func InferRoomType(task models.TeachingTask, course models.Course) string {
    // 1. 显式指定 — 最高优先级
    if task.RequiredRoomType != "" {
        return task.RequiredRoomType
    }
    // 2. Course.Category 映射
    if course.Category != "" {
        if rt, ok := models.CategoryRoomTypeMap[course.Category]; ok && rt != "" {
            return rt
        }
    }
    // 3. 名称推断 — Deprecated, v0.6 将移除
    //    详见 §5 Deprecated 路径退出计划
    return inferRoomTypeByName(course.Name)
}

// inferRoomTypeByName 基于课程名关键字推断教室类型。
//
// Deprecated: v0.5.3 保留作为 fallback，v0.6 将通过 migration 补全 Category 后移除。
// 新代码不应依赖此函数，应通过 Course.Category 或 task.RequiredRoomType 指定。
func inferRoomTypeByName(courseName string) string {
    if models.IsSportsCourse(courseName) {
        return models.RoomTypeGymnasium
    }
    if IsLabCourse(courseName) {
        return models.RoomTypeLab
    }
    if IsComputerCourse(courseName) {
        return models.RoomTypeComputer
    }
    return ""
}

// IsSharedVenue 判断教室是否为共享场地（允许多课程并发使用）。
func IsSharedVenue(room models.Classroom) bool {
    return models.SharedVenueTypes[room.Type]
}

// AllowedRooms 从候选列表中筛选出所有匹配的教室。
// 用于 OR-Tools 预计算：Go 侧调用此函数，将结果序列化到 JSON payload。
// Python 侧只读预计算结果，不做任何匹配逻辑。
func AllowedRooms(task models.TeachingTask, course models.Course, rooms []models.Classroom) []models.Classroom {
    var out []models.Classroom
    for _, room := range rooms {
        if Match(task, course, room).OK {
            out = append(out, room)
        }
    }
    return out
}

// ExplainRequirement 生成任务的资源需求描述（纯结构化数据）。
// 用于排课诊断报告、调课失败提示、日志输出 — 不做匹配判断，只描述需求。
// 调用方自行组装展示文案。
func ExplainRequirement(task models.TeachingTask, course models.Course) ResourceRequirement {
    reqType := InferRoomType(task, course)
    source := "none"
    if task.RequiredRoomType != "" {
        source = "explicit"
    } else if course.Category != "" {
        if _, ok := models.CategoryRoomTypeMap[course.Category]; ok {
            source = "category"
        }
    } else if reqType != "" {
        source = "name_fallback"
    }

    reqEquip := ParseEquipment(task.RequiredEquipment)
    var equipList []string
    for item := range reqEquip.items {
        equipList = append(equipList, item)
    }

    isShared := false
    if reqType != "" {
        isShared = models.SharedVenueTypes[reqType]
    }

    return ResourceRequirement{
        RoomType:       reqType,
        RoomTypeSource: source,
        Equipment:      equipList,
        IsShared:       isShared,
    }
}

// ExplainMismatch 将 MatchResult 转为人类可读的 mismatch 描述。
// 用于调课失败提示、排课诊断日志 — 根据 Code 分类生成文案。
func ExplainMismatch(result MatchResult) string {
    if result.OK {
        return ""
    }
    switch result.Code {
    case CodeRoomTypeMismatch:
        return fmt.Sprintf("教室类型不匹配: 需要%s，当前为%s", result.RequiredType, result.ActualType)
    case CodeSpecialtyExclusion:
        return fmt.Sprintf("排他教室限制: 普通课程不能使用%s", result.ActualType)
    case CodeEquipmentMissing:
        return fmt.Sprintf("缺少设备: %s", strings.Join(result.MissingEquipment, ", "))
    default:
        return result.Reason
    }
}
```

### 2.3 数据流（修订：OR-Tools 不做推断）

```
Course.Category (新)     ──┐
Classroom.RoomType (现有)  ──┤
Classroom.Equipment (新)  ──┤
TeachingTask.ReqRoomType ──┤──→ Match(task, course, room) → MatchResult
TeachingTask.ReqEquipment──┘    │
                                ├─→ SA: 过滤候选教室
                                ├─→ AllowedRooms() → OR-Tools JSON payload (预计算列表)
                                │   Python 侧: 只读 allowedRoomIds，不做匹配
                                ├─→ MoveService: CheckMove 校验 + Reason 作为冲突描述
                                └─→ ExplainRequirement() → 排课诊断报告 / 调课失败提示 / 日志输出
                                    ExplainMismatch(result) → 人类可读的 mismatch 描述
```

**关键变化**：OR-Tools 不再自己做匹配。Go 侧通过 `AllowedRooms()` 预计算每个 task 的可用教室列表，序列化到 JSON。Python 侧只读列表。

- 现有课程保持 Category="" → 名称推断 fallback（Deprecated 但可用）
- 新增课程在录入时选择 Category
- 可批量通过 Excel 导入补全 Category 字段

## 10. 决策记录

| # | 决策 | 理由 |
|---|------|------|
| 1 | Category 放 Course 而非 TeachingTask | 课程类别是课程固有属性，多个教学任务共享 |
| 2 | RequiredRoomType 放 TeachingTask | 同一课程不同任务可能需要不同教室（分班实验 vs 理论讲授） |
| 3 | Equipment 用 JSON string + EquipmentSet 封装 | 简化存储；EquipmentSet 提供语义化查询，避免散落字符串比较 |
| 4 | 保留名称推断 fallback 但标记 Deprecated | 旧数据零迁移成本；v0.6 提供迁移路径后移除 |
| 5 | Matcher 为包级纯函数而非 struct | 无状态 = 可并发 = 可测试 = 无副作用 |
| 6 | Match() 返回 MatchResult 而非 bool | 诊断信息一次生成，调用方直接复用，避免 bool→再判断的二次开销 |
| 7 | RoomType 用 const 集中定义 | 新增类型只改一处，不碰匹配逻辑 |
| 8 | OR-Tools 由 Go 预计算 AllowedRooms | 单一真相源；Python 不做推断；消除两端不同步风险 |
| 9 | MoveService 直接调 Match() | 不自建任何判断逻辑，避免第三个匹配实现 |
| 10 | ExplainRequirement() 返回纯结构化 ResourceRequirement | 为排课诊断、调课提示、日志预留结构化接口；不含展示文案，调用方自行组装 |
| 11 | ExplainMismatch(result) 根据Code分类生成文案 | 集中 mismatch 描述逻辑，避免各调用方重复写 switch |
| 12 | ResourceMatchCode 用 int 枚举 | switch 友好，日志友好，避免拼写错误；产品定位为教务软件，不需要 string 可读性 |
