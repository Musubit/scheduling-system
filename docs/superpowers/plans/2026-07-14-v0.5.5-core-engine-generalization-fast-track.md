# v0.5.5 核心引擎通用能力（TIME_ONLY）快速落地计划

> **面向 AI 代理的工作者：** 建议使用 superpowers:subagent-driven-development 按任务执行，并在每个任务后进行最小回归验证。

**目标：** 在 3-5 天内交付“关闭教室场地分配后仍可完成课程安排”的可发布能力，作为 v0.5.5 的最高优先级门禁。

**架构：** 以 `SchedulingMode` 作为单一模式真值源，Orchestrator 按模式装配阶段：`FULL_SCHEDULING` 走时间+教室两阶段，`TIME_ONLY_SCHEDULING` 仅走时间阶段；评分维度与结果展示随模式自适应。

**技术栈：** Go（services + scheduling/types）、Vue3/Pinia、Wails bindings。

---

## 交付边界

### 必做（P0）

1. 后端支持 `SchedulingConfig.Mode` 并进入真实控制流。
2. TIME_ONLY 模式可完成排课，且不生成当前版本的教室分配记录。
3. 评分维度按模式生效（TIME_ONLY 下资源维度禁用）。
4. 前端新增并透传“关闭教室场地分配”开关。
5. 双模式最小回归通过。

### 不做（本计划范围外）

1. AcademicTerm / TeachingWeek 新表与日历扩展。
2. Solver 按 TermType 感知。
3. H3 手动调课后保存快照前端闭环。

## 完成定义（DoD）

1. 用户可在 UI 一键切换 FULL / TIME_ONLY。
2. TIME_ONLY 下不因教室不足导致整体排课失败。
3. TIME_ONLY 评分与展示语义自洽，不出现伪资源分。
4. FULL 路径关键行为无回归。
5. 文档与版本状态同步更新。

## 风险与缓解

1. 风险：模式逻辑分散，导致行为不一致。
   缓解：以 Orchestrator 装配为唯一模式分发点。

2. 风险：历史快照读取与新分桶不兼容。
   缓解：保留旧字段兼容读取，新增字段走可选语义。

3. 风险：前端展示仍假定资源维度必有。
   缓解：结果渲染以 mode-aware 条件分支为准，补充视图单测/手测。
