# 领域术语表

高校排课系统的统一领域语言。

## 核心实体

| 术语 | 英文 | 定义 |
|---|---|---|
| **课程** | Course | 培养方案中的一门课，如"数据结构"。有学时、所属院系。 |
| **教师** | Teacher | 授课人员。具有时间偏好（避免早课/晚课）、到校天数限制、低楼层偏好。 |
| **教室** | Classroom | 物理教学场所。有楼栋、楼层、容量。 |
| **班级** | ClassGroup | 学生行政班，如"计算机2301"。同一班级同一时间不能上两门课。 |
| **学期** | Semester | 排课的时间范围，如"2025-2026 第二学期"。 |

## 排课概念

| 术语 | 英文 | 定义 |
|---|---|---|
| **课表条目** | ScheduleEntry | 一次排课安排：某课程在某时间、某教室、由某教师授课。 |
| **节次** | Period | 一天中的第N节课。0=第1节，10=第11节。有效起始节次：0/2/4/6/8（第1/3/5/7/9节）。 |
| **星期** | DayOfWeek | 0=周一，6=周日。内部使用0-based，显示时+1。 |
| **锁死时段** | LockedTimeSlot | 全校不可排课的时间段，如"周四下午5-6节（党会）"。 |
| **院系范围** | Scope | 按院系筛选排课范围，可全校或指定学院。 |

## 约束分类

### 硬约束（不可违反）

| 约束 | 说明 |
|---|---|
| 教师时间冲突 | 同一教师同一时间不能上两门课 |
| 教室占用冲突 | 同一教室同一时间不能被两门课占用 |
| 班级时间冲突 | 同一班级同一时间不能上两门课 |
| 锁死时段规避 | 不可在全局锁定时间段内排课 |

### 软约束（尽量满足，可权衡）

| 约束 | 说明 |
|---|---|
| 教师偏好-早课 | 教师设置"避免早课"后，尽量不排1-2节 |
| 教师偏好-晚课 | 教师设置"避免晚课"后，尽量不排7-8节及以后 |
| 课程均匀分布 | 同一课程的各次安排尽量分散在一周不同天 |
| 到校天数限制 | 教师每周到校天数尽量不超过设定值（默认3天） |
| 低楼层优先 | 有"优先低楼层"偏好的教师尽量排在低楼层教室 |
| 容量适配 | 教室容量应不小于班级人数（软约束，允许例外） |

## 排课流程

| 术语 | 定义 |
|---|---|
| **自动排课** | 运行模拟退火求解器，自动生成课表方案 |
| **微调** | 排课后手动拖拽或编辑个别课表条目，系统前置校验硬约束 |
| **CheckMove** | 微调前的校验API：检查移动是否违反硬约束，违反则拒绝 |
| **验证报告** | 排课或微调后的评估产出：硬约束合规状态 + 软约束评分明细 |

## 快照

| 术语 | 定义 |
|---|---|
| **快照** | 某个时刻排课方案的评估记录。自动快照：排课完成后生成。手动快照：微调后用户主动生成。 |
| **快照明细** | 快照中每个教师的评分贡献，用于"拖后腿排行榜"。 |

## 评分

| 术语 | 定义 |
|---|---|
| **综合评分** | 0-100分，四项软约束的加权和。默认均权（各25分）。 |
| **评分明细** | 按约束维度拆分的得分：教师偏好/课程间隔/到校天数/低楼层。 |

## 求解器

| 术语 | 定义 |
|---|---|
| **模拟退火** | Simulated Annealing（SA）。纯Go实现，零依赖。贪心初始解 + 邻域搜索（移动/交换）+ 温度冷却。 |
