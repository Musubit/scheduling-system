# ADR-0004: TeachingTask 实体替代院系模糊匹配

## 状态

已采纳

## 上下文

原排课系统的课程-教师-班级关联是**隐式的**：求解时通过硬编码的 `deptMap` 按院系名称模糊匹配。例如，`dept="sci"` 的课程只能匹配 `dept="理学院"` 的教师和班级。这种设计有三个根本缺陷：

1. **无法表达真实教学安排**：实际场景中，"高等数学A"可能由张三教计科1/2班（合班）、李四教软工1/2班（合班）、王五教大数据1班（单班）。同名课程存在多个平行教学班，每个教学班有明确的教师-班级绑定。

2. **跨院系合班无法支持**：例如"大学英语"课程，教师来自外语学院，班级可能来自计算机学院和机械学院。院系匹配逻辑禁止这种跨院系关联。

3. **数据冗余和脆弱性**：`deptMap` 和 `reverseDeptMap` 硬编码 19 个院系的中英文名称映射，新增院系或改名都会破坏匹配逻辑。

## 决策

**引入 `TeachingTask` 实体，用显式关联替代院系模糊匹配。**

### 数据模型

```
TeachingTask {
    CourseID   FK   // 课程
    TeacherID  FK   // 教师（必选，不再由求解器分配）
    SemesterID FK   // 学期
}

TeachingTaskClass {
    TeachingTaskID FK  // 教学任务
    ClassGroupID   FK  // 班级（可多选，支持合班）
}
```

### 求解器适配

- `ScheduleEntry.ClassGroupID` 保留但弱化，新增 `ScheduleEntry.TeachingTaskID`
- 求解器 `buildInitial()` 改为迭代 TeachingTask 列表，教师和班级直接读取关联，不再做院系匹配
- 合班支持：教学任务内所有班级同时占用/释放，硬约束检查整体判断
- `deptMap` 和 `reverseDeptMap` 从求解逻辑中移除（仅保留用于排课范围的院系筛选）

### 前端支持

- 资源管理页新增"教学任务"Tab，支持手动绑定 + Excel 导入
- 智能合班检测：扫描同课程名+同教师的教学任务，自动建议合并

## 理由

1. **语义准确**：TeachingTask 直接表达"谁教谁"的真实教学安排，消除院系匹配的歧义
2. **支持合班**：TeachingTaskClass 多对多关联天然支持合班上课
3. **数据完整性**：教师和班级在排课前就已绑定，求解器不再"猜"关联，结果更可控
4. **可维护性**：不再依赖硬编码的院系名称映射，新增/改名院系无影响
5. **向后兼容**：ScheduleEntry 保留 ClassGroupID，旧数据不受影响

## 后果

- **正面**：领域模型更准确，支持合班，消除关联歧义，提高排课成功率
- **负面**：新增两张表和一个管理界面，用户需在排课前创建教学任务（可通过 Excel 批量导入降低负担）
- **可逆性**：TeachingTaskID 为可空字段，若回退可恢复纯 ClassGroupID 模式
