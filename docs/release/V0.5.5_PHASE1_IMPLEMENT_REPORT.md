## 三、模型变化对照

### 3.1 Semester

**v0.5.4（旧）**

```go
type Semester struct {
    gorm.Model
    Name      string   // "2025-2026 第二学期"
    IsActive  bool
    StartDate string   // "2025-09-01"
}
```

**v0.5.5（新）**

```go
type Semester struct {
    gorm.Model
    AcademicYear string    `gorm:"uniqueIndex:idx_semester;size:20;not null"`  // "2025-2026"
    Term         string    `gorm:"uniqueIndex:idx_semester;size:10;not null"`  // FIRST/SECOND
    StartDate    time.Time
    EndDate      time.Time
    Status       string    `gorm:"size:20;default:active"`  // active/archived/planned
}
// 派生显示名：DisplayName() string
```

### 3.2 ScheduleEntry

**旧**
```go
Semester string `gorm:"size:50;uniqueIndex:idx_schedule_room"`
```

**新**
```go
SemesterID uint `gorm:"uniqueIndex:idx_schedule_room;not null"`
Semester   Semester `gorm:"foreignKey:SemesterID"`
```

### 3.3 ScheduleSnapshot

**旧**：`Semester string`
**新**：`SemesterID uint`

## 五、风险

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 已投产数据库 schema 变更（`ScheduleEntry.Semester` 列类型变） | — | — | 无生产数据，v0.5.5 起要求删除旧 db 重新初始化 |
| 前端仍引用 `s.name` / `s.isActive`（`SettingsPage.vue`、`AppToolbar.vue`、`SchedulingPage.vue` 等） | 中 | 运行时空字符串或 undefined | **Phase 3 清理**——本 Phase 明确不做，构建可通过（TypeScript 未标记为 required 属性） |
| Wails bindings 由 `.ts` 变 `.js` 又变回 `.ts` | 低 | 已通过 `wails3 generate bindings -ts` 恢复 `.ts` | 已在提交前清理残留 `.js` 副本 |
| `snapshotservice.CreateManualSnapshot` 现在需要 `semesterID uint`，旧前端调用可能仍传字符串 | 中 | Wails 类型层会阻挡编译，但如有运行时 RPC 需重新校验 | Bindings 已同步，前端类型系统会在下次构建时暴露不兼容 |
| 前端 store（`app.ts` / `schedule.ts`）中 `semesters` 类型迁移未做 | 低 | 页面显示可能异常但不阻塞排课逻辑 | 由 Phase 3 承接 |

## 七、遗留观察（非本 Phase 处理）

1. **前端 `s.name` / `s.isActive` 15+ 处引用**：`SettingsPage.vue`、`AppToolbar.vue`、`SchedulingPage.vue`、`HistoryComparePage.vue`、`stores/app.ts` 等。由于 TypeScript 类型定义中所有属性为可选/宽松，build 通过；但运行时会读到 undefined，需 Phase 3 集中改造。
2. **`SettingsPage.vue` 编辑学期表单仍绑定 `isActive`**：需要 Phase 3 用 `Status` 单选替代。
3. **Wails Bindings 目录中的 `.js` 副本**：Phase 1 中已删除（`find frontend/bindings -type f -name "*.js" -delete`）。后续 `wails3 generate bindings` 若不带 `-ts` 会再生成 `.js`——建议在构建脚本或 CI 中固定 `-ts` 参数。
