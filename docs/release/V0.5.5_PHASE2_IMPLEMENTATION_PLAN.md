## 一、Exact Files to Add（6 个 + 1 个可选）

### 1.1 Model 层（1 文件）

| 文件 | 内容 |
|---|---|
| `backend/models/academic_term.go` | `AcademicTerm` struct + `Season` / `TermType` 常量 + 派生方法 |

**结构设计**（对齐 Phase 1 Semester 风格：`gorm.Model` + tag + 常量集中）：

```
type AcademicTerm struct {
    gorm.Model
    SemesterID uint      `gorm:"uniqueIndex:idx_term;index;not null" json:"semesterId"`
    Season     string    `gorm:"uniqueIndex:idx_term;size:10;not null" json:"season"`
    TermType   string    `gorm:"uniqueIndex:idx_term;size:20;not null" json:"termType"`
    StartWeek  int       `gorm:"not null" json:"startWeek"`
    EndWeek    int       `gorm:"not null" json:"endWeek"`
    Semester   Semester  `gorm:"foreignKey:SemesterID" json:"semester,omitempty"`
}
```

**Season 常量**（对齐 `SemesterTermFirst/Second` 命名习惯）：

```
const (
    SeasonAutumn = "AUTUMN"
    SeasonWinter = "WINTER"
    SeasonSpring = "SPRING"
    SeasonSummer = "SUMMER"
)
```

**TermType 常量**：

```
const (
    TermTypeTeaching = "TEACHING"
    TermTypeExam     = "EXAM"
    TermTypePractice = "PRACTICE"
)
```

**纯 Go 派生方法**（无 DB 依赖）：

- `(a AcademicTerm) IsExamPeriod() bool`
- `(a AcademicTerm) IsTeachingPeriod() bool`
- `(a AcademicTerm) IsPracticePeriod() bool`
- `(a AcademicTerm) WeekCount() int`（`EndWeek - StartWeek + 1`）
- `(a AcademicTerm) ContainsWeek(w int) bool`（`w >= StartWeek && w <= EndWeek`）

### 1.2 领域包（3 文件）

| 文件 | 内容 |
|---|---|
| `backend/services/academic_calendar/types.go` | `TeachingWeekView` ViewModel struct |
| `backend/services/academic_calendar/calendar.go` | 4 个纯函数：`DateOfWeek` / `WeeksOfSemester` / `CurrentSemester` / `CurrentWeek` |
| `backend/services/academic_calendar/validator.go` | `ValidateAcademicTerm(sem, []terms) error` |

**TeachingWeekView 结构**：

```
type TeachingWeekView struct {
    WeekNumber int
    StartDate  time.Time // 该周周一
    EndDate    time.Time // 该周周日
    TermID     uint      // 所属 AcademicTerm.ID
    Season     string
    TermType   string
}
```

**函数签名**（严格无 DB 依赖，输入即 model 值 + `time.Time`）：

```
package academic_calendar

// DateOfWeek 返回第 week 周周一的日期（0:00 UTC）。
// 若 week < 1 或 sem.StartDate 为零值，返回 sem.StartDate 原值。
func DateOfWeek(sem models.Semester, week int) time.Time

// WeeksOfSemester 返回学期完整周次视图。
// 若 terms 为空，退化为空列表；正常情况按 max(EndWeek) 生成 1..N。
func WeeksOfSemester(sem models.Semester, terms []models.AcademicTerm) []TeachingWeekView

// CurrentSemester 依据 now 归属活跃学期（now ∈ [StartDate, EndDate]）。
// 若 now 落学期外（暑寒假），返回 nil；调用方决定回退策略。
func CurrentSemester(semesters []models.Semester, now time.Time) *models.Semester

// CurrentWeek 返回 now 相对 sem.StartDate 的周次编号（1-based）。
// 若 now 早于 sem.StartDate，返回 0；若晚于 sem.EndDate 或超出 max，返回 max。
func CurrentWeek(sem models.Semester, now time.Time) int

// ValidateAcademicTerm 检验一组 AcademicTerm 的 Week 区间：
//   - 每段 StartWeek >= 1 且 <= EndWeek（否则 bound error）
//   - 段间无 overlap
//   - 覆盖 1..max（无 gap）
// 返回聚合的多错误（errors.Join）；调用方决定是否阻塞。
func ValidateAcademicTerm(sem models.Semester, terms []models.AcademicTerm) error
```

## 五、Risk Control

### 5.1 Semester Unchanged 保障

| 保障机制 | 说明 |
|---|---|
| 代码层面 | `academic_term.go` 通过 `SemesterID uint` FK + `Semester Semester` Association 单向引用，不修改 Semester struct |
| Seed 层面 | `seedAcademicTerms` 只读 Semester（`db.Order("id asc").Find(&semesters)`），只写 AcademicTerm |
| 测试层面 | `academic_calendar` 函数以 `models.Semester` 为**只读输入**，无 DB 副作用 |
| 验证 | NC1 `git diff` 校验 `models/setting.go` 未变 |

### 5.2 Solver Unchanged 保障

| 保障机制 | 说明 |
|---|---|
| 不改 solver 入口 | `scheduling_service.go` / `sa_solver.go` / `ortools_client.go` / `solver.py` 全部不动 |
| 不引入 TermType 感知 | AcademicTerm 数据存在但 SA / OR-Tools 无消费点 |
| 既有 SA 测试保护 | `sa_bench_test.go` / `sa_delta_parity_test.go` / `sa_solver_test.go` / `sa_postopt_test.go` / `score_completeness_test.go` 均保留，AC7 校验全绿 |
| 验证 | NC3 `git diff` 校验 |

### 5.3 Frontend Unchanged 保障

| 保障机制 | 说明 |
|---|---|
| 不动 `frontend/src/**` | 所有前端文件保持原样，包括残留 `s.name` / `s.isActive`（Phase 3 承接） |
| `npm run build` 门禁 | AC8 保证前端 build 通过，反证前端未受影响 |
| 验证 | NC2 `git diff` 校验 |

### 5.4 Wails Binding Unchanged 保障

| 保障机制 | 说明 |
|---|---|
| Phase 2 不暴露新 service | `academic_calendar_service.go` 明确延后 Phase 3 |
| 不重生成 bindings | 不运行 `wails3 generate bindings`（Phase 2 无必要） |
| 现有 `frontend/bindings/` 工作副本差异不进入本 Phase | commit 时明确 `git add` 白名单，不 stage `frontend/bindings/**` |
| 验证 | NC5 `git diff` 校验 |

### 5.5 其他风险

| 风险 | 处置 |
|---|---|
| Seed 幂等错误导致每次运行都插入 | `FirstOrCreate` + 复合键（对齐 Phase 1 Semester Seed 模式），单测 + 手工连续运行 3 次验证 |
| `ValidateAcademicTerm` 误报正常数据 | Seed 数据必须通过 `TestValidateAcademicTerm_OK` 才允许合入 |
| `CurrentSemester` 空列表 panic | 函数首行 `if len(semesters) == 0 { return nil }` 明确降级；单测覆盖 |
| SQLite 复合唯一索引 tag 拼写错误 | 对齐 `Semester.AcademicYear` / `Term` 的 `uniqueIndex:idx_semester` 命名约定，AutoMigrate 时 GORM 校验索引存在 |
