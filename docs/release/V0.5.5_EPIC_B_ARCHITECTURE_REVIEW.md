## 一、当前 Semester 模型分析

### 1.1 模型定义

**位置**：`backend/models/setting.go:6-11`

```go
type Semester struct {
    gorm.Model
    Name      string // "2025-2026 第二学期"  — 显示名 + 唯一键 + 查找键
    IsActive  bool   // 布尔开关 — 手动切换
    StartDate string // "2025-09-01" — 字符串，非 time.Time
}
```

### 1.2 字段职责（当前）

| 字段 | 职责 | 问题 |
|---|---|---|
| `Name` | 三重用途：显示标签、唯一约束、查找键（`Where("name = ?")`） | 改名即破坏所有引用 |
| `IsActive` | "当前学期"标志 | 只能表达一个学期为 active；不能表达"按时间自动判定" |
| `StartDate` | 学期起始日期锚点 | `string` 类型，不可做日期运算 |

### 1.3 生命周期

```
Seed 创建 → 用户手动切换 IsActive → 排课引用 → 版本归档 → 无删除
```

无"开始日期/结束日期自动判定当前学期"的能力，无"学年"概念。

## 二、全链路 Semester 使用位置

### 2.1 核心发现：两套 Semester 引用系统并存

| 引用方式 | 使用方 | 关联类型 |
|---|---|---|
| `SemesterID uint` (FK) | `TeachingTask`, `ScheduleVersion` | 健康 |
| `Semester string` | `ScheduleEntry`, `ScheduleSnapshot` | 🔴 高危 |

**示例**：

```go
// schedule_entry.go:13
Semester string `gorm:"size:50;uniqueIndex:idx_schedule_room" json:"semester"`

// snapshot.go:15
Semester string `gorm:"index;size:50" json:"semester"`
```

`ScheduleEntry` 的唯一索引 `idx_schedule_room` 依赖此字符串字段。`MoveService.computeScore`、`SnapshotService`、`VersionService` 全部通过字符串匹配学期。

### 2.2 依赖矩阵

| 文件 | 使用 Semester 的方式 | 变更风险 |
|---|---|---|
| `models/teaching_task.go` | `SemesterID uint` FK → `semesters.id` | 🟢 低（已是 FK，仅需数据迁移） |
| `models/schedule_entry.go` | `Semester string` + 唯一索引 | 🔴 高（需改为 FK，索引重建） |
| `models/snapshot.go` | `Semester string` + 索引 | 🔴 高（同左） |
| `models/schedule_version.go` | `SemesterID uint` FK | 🟢 低 |
| `services/resource_service.go` | `GetSemesters()` / `GetActiveSemester()` / `CreateSemester` / `UpdateSemester` / `DeleteSemester` | 🔴 高（CRUD 全部重写，IsActive 移除） |
| `services/scheduling_service.go` | `SchedulingConfig.Semester string` + `SemesterID uint` 双重 | 🟡 中（字符串删除，统一用 ID） |
| `services/move_service.go` | `entry.Semester` 字符串 → `computeScore` | 🟡 中（改为 ID） |
| `services/snapshot_service.go` | `snapshot.Semester` 字符串查询 | 🟡 中（改为 ID） |
| `services/version_service.go` | `resolveSemesterID(name)` 名字→ID 转换 | 🔴 高（函数删除，改为直传 ID） |
| `services/sa_initial.go` | `ctx.semester` 字符串 | 🟡 中（改为 ID） |
| `database/seed.go` | `Semester{Name, IsActive, StartDate}` 写入；`InterpretSemester` 硬编码 `"2025-2026 第二学期"` | 🔴 高（seed 完全重写） |
| `frontend/src/stores/app.ts` | `semesters: {ID, name, isActive}` | 🟡 中（类型更新，GetActiveSemester 替换） |
| `frontend/src/stores/schedule.ts` | `currentWeek = ref(8)` 硬编码；`nextWeek: < 20` 硬编码 | 🟡 中（改为后端 API） |
| `frontend/src/components/schedule/WeekView.vue` | `startOfYear + (currentWeek-1)*7` 伪日期 | 🔴 高（整段删除，改为消费后端数据） |
| `frontend/src/components/layout/AppToolbar.vue` | `s.name` 学期下拉 | 🟢 低（Name 改为派生，保持 ID 作为 value） |

## 三、AcademicYear / Term / TeachingWeek / Holiday 必要性分析

### 3.1 AcademicYear（学年）

| 维度 | 分析 |
|---|---|
| 当前状态 | 不存在。`"2025-2026"` 隐藏在 `Name` 字符串中 |
| 必要性 | **高**。湖工大校历以"学年"为组织单位，两学期属于同一学年。需结构化字段支持按学年筛选、排序 |
| 建议 | `Semester.AcademicYear string`，Validator 校验 `^\d{4}-\d{4}$`。暂不独立建表 |
| 优先级 | Phase 1 |

### 3.2 Term（学期序数）

| 维度 | 分析 |
|---|---|
| 当前状态 | 隐藏在 `Name` 中（"第一学期"/"第二学期"） |
| 必要性 | **高**。需结构化枚举 `FIRST / SECOND` 支持学年内排序、筛选 |
| 建议 | `Semester.Term string`，枚举 `FIRST / SECOND` |
| 优先级 | Phase 1 |

### 3.3 TeachingWeek（教学周）

| 维度 | 分析 |
|---|---|
| 当前状态 | 不存在。前端硬编码 `WeekView.vue:85-95` 用"当年 1 月 1 日 + week×7"伪计算，`schedule.ts:8` 硬编码 `currentWeek = ref(8)` |
| 必要性 | **高**。后端是唯一时间源。日期由 `Semester.StartDate + (WeekNumber-1)*7` 派生 |
| 建议 | **不建表**。作为 ViewModel 由 `services/academic_calendar/` 派生。`Semester.TeachingWeeks int` 作为跨表锚点 |
| 优先级 | Phase 2 |

### 3.4 AcademicTerm（学段）

| 维度 | 分析 |
|---|---|
| 当前状态 | 不存在。Solver 不区分"秋季常规段"和"冬季实践段"，所有周次一视同仁 |
| 必要性 | **高**。湖工大校历有"秋季/冬季"和"常规/考试/实践"两层分类。需表达不同 Term 的能力 |
| 建议 | 新建 `academic_terms` 表。`Season(AUTUMN/WINTER/SPRING/SUMMER)` + `TermType(TEACHING/EXAM/PRACTICE)`。能力字段由 TermType 派生（不落库） |
| 优先级 | Phase 2 |

### 3.5 Holiday（节假日）

| 维度 | 分析 |
|---|---|
| 必要性 | **低**。湖工大实际运行中节假日由学校临时调整，排课系统不应强制阻止 |
| 建议 | **不引入**。独立 Epic「Academic Calendar Exception」承接 |
| 优先级 | 不纳入 v0.5.5 |

## 四、模型问题 → 目标模型 → 迁移策略

### 4.1 当前模型问题（5 个核心技术债）

| # | 问题 | 严重性 | 影响 |
|---|---|---|---|
| P1 | `ScheduleEntry.Semester` 存为字符串而非 FK | 🔴 | 唯一索引依赖字符串，不可改名 |
| P2 | `Semester.Name` 三重用途（显示+唯一+查找） | 🔴 | 改名即破坏 `version_service`、`snapshot_service` 等 |
| P3 | `Semester.StartDate` 为 `string` 而非 `time.Time` | 🟡 | 前端被迫伪计算 Week 日期 |
| P4 | 前端 WeekView 硬编码日期计算 | 🔴 | 与 Semester.StartDate 无关，完全错误 |
| P5 | 无 AcademicTerm 概念 | 🟡 | Solver 不区分考试周/实践周 |

### 4.2 目标模型

```
┌──────────────────────┐
│      Semester        │  ← 表名 semesters 不变
├──────────────────────┤
│ AcademicYear string  │  ← "2025-2026"（Validator: ^\d{4}-\d{4}$）
│ Term        string  │  ← FIRST / SECOND
│ StartDate   time.Time│  ← 改为 time.Time，强制周一
│ TeachingWeeks int    │  ← 跨表锚点（如 21）
└──────┬───────────────┘
       │ 1:N
       ▼
┌──────────────────────┐
│   AcademicTerm       │  ← 新表
├──────────────────────┤
│ SemesterID  uint     │
│ Season      string  │  ← AUTUMN / WINTER / SPRING / SUMMER
│ TermType    string  │  ← TEACHING / EXAM / PRACTICE
│ StartWeek   int     │  ← Semester 内连续编号
│ EndWeek     int     │
│ UNIQUE (SemesterID, Season, TermType)
└──────────────────────┘

TeachingWeek — 不建表，派生视图
```

**删除字段**：`Name`（派生）、`IsActive`（派生）

**派生规则**：
- `Name` = `AcademicYear + termLabel(Term)`  → 例 "2025-2026第一学期"
- `IsActive` = `CurrentSemester(semesters, now)` → 按时间自动判定
- `EnableCourseScheduling()` = `TermType == "TEACHING"` → Go 方法
- `Week.StartDate` = `Semester.StartDate + (WeekNumber-1)*7` → 纯函数

### 4.3 迁移策略

| 阶段 | 内容 | 破坏性 |
|---|---|---|
| Phase 1 | Semester 模型重构 + ScheduleEntry/Snapshot 改 FK | 🔴 破坏性（Schema 变更） |
| Phase 2 | AcademicTerm 新增 + services/academic_calendar/ 建设 | 🟢 增量 |
| Phase 3 | 前端硬编码清除 + Solver 接入 Term 能力 | 🟡 联动 |

**数据迁移**：无生产数据，可直接 `DROP TABLE semesters` + 重建。`schedule_entries` 和 `schedule_snapshots` 同理。

### 4.4 风险

| 风险 | 概率 | 缓解 |
|---|---|---|
| SQLite AutoMigrate 不自动重建复合唯一索引 | 中 | `schedule_entries` DROP + 重建 |
| `version_service.go` 的 `resolveSemesterID(name)` 依赖 Name 字符串 | 高 | 整函数删除，调用方改为传 SemesterID |
| 前端 `semesterFilter` 字符串代理仍然依赖 s.name | 高 | 改为派生计算 |
| Wails binding 类型变动导致前端编译失败 | 低 | 重生成 bindings + 全量构建验证 |

## 七、与新模型对齐的 API Contract

### 后端 API（Wails binding）

```
GET /api/semesters
  → [{id, academicYear, term, startDate, teachingWeeks}]

GET /api/semesters/current     (派生: CurrentSemester(now))
  → {id, ...}

GET /api/semesters/:id/weeks   (派生: WeeksOfSemester)
  → [{weekNumber, startDate, endDate, termId, capabilities}]

GET /api/semesters/:id/current-week  (派生: CurrentWeek)
  → {weekNumber, termId}
```

### Solver 输入（JSON）

当前 solver 输入维持不变（`StartWeek/EndWeek` int）。未来可扩展 `AcademicTermContext` 数组使 solver 按 TermType 智能排课。
